HTTP Security Headers CVSS 4.3

Attack Scenario: Attacker exploits HTTP Security Headers at https://www.caricite.com/ using: Missing: content-security-policy

Business Impact: Potential data breach, system compromise, and reputational damage.

Remediation:

  • Add Content-Security-Policy: default-src 'self'
  • Add X-Frame-Options: DENY
  • Add X-Content-Type-Options: nosniff
  • Add Strict-Transport-Security: max-age=31536000; includeSubDomains
  • Remove Server, X-Powered-By headers (hide tech stack)

Code Fix:

@app.after_request
def add_security_headers(resp):
    resp.headers['Content-Security-Policy'] = "default-src 'self'"
    resp.headers['X-Frame-Options'] = 'DENY'
    resp.headers['X-Content-Type-Options'] = 'nosniff'
    resp.headers['Strict-Transport-Security'] = 'max-age=31536000; includeSubDomains'
    return resp
