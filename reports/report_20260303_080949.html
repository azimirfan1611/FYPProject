Authentication CVSS 7.5

Attack Scenario: Attacker exploits Authentication at http://webapp:5000/login using payload: Valid credentials found: admin/admin

Business Impact: Potential data breach, system compromise, and reputational damage.

Remediation:

  • Implement account lockout after 5 failed attempts
  • Enforce strong password policy (min 12 chars, complexity)
  • Use bcrypt/argon2 for password hashing — never store plaintext
  • Regenerate session ID after successful login (prevent fixation)

Path Traversal CVSS 7.5

Attack Scenario: Attacker exploits Path Traversal at http://webapp:5000/files using payload: ../../proc/self/environ

Business Impact: Potential data breach, system compromise, and reputational damage.

Remediation:

  • Resolve canonical path and verify it starts with the allowed base directory
  • Use a whitelist of permitted filenames
  • Never pass user input directly to filesystem APIs

Command Injection CVSS 9.8

Attack Scenario: Attacker exploits Command Injection at http://webapp:5000/ping using payload: 127.0.0.1; id

Business Impact: Potential data breach, system compromise, and reputational damage.

Remediation:

  • Never pass user input to shell commands
  • Use language-native APIs (e.g., socket libraries) instead of ping
  • If shell is required, use allowlist validation on input

IDOR / Broken Access Control CVSS 7.5

Attack Scenario: Attacker exploits IDOR / Broken Access Control at http://webapp:5000/profile/1 using payload: Accessed profile of user id=1 as alice

Business Impact: Potential data breach, system compromise, and reputational damage.

Remediation:

  • Implement server-side ownership checks on every resource access
  • Use indirect reference maps (random tokens) instead of sequential IDs
  • Apply authentication middleware to all sensitive API endpoints
